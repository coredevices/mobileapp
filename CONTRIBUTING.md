### Contributing code to libpebble3

Everyone is welcome to contribute code to libpebble3, provided that they are willing to license their contributions to Core Devices under a [Contributor License Agreement](https://cla-assistant.io/coredevices/libpebble3) (CLA). This ensures that their contribution will be made available under an OSI-approved open-source license, currently licensed under GNU GPLv3.

### AI

If you use AI to write the code that you are submitted, you must mention this in the PR description.

We do not accept "vibe-coded" PRs - i.e. you can use AI to assist you, but you must have an understanding of the code that you are changing (AI is good at helping you get this!), and you must have reviewed the changes yourself before submitting them.

If we don't review/accept your PR, please understand that we have extremely limited resources - to merge your change requires us to review it, possibly test it with real hardware, but then most importantly to support/maintain it in the future. It might feel like it is helpful to implement a missing feature which we haven't gotten to yet - but (especially if it is largely vibe-coded and undirected to make the right architectural choices) this can often result in more work than waiting until we get around to the feature.

Please read https://tombedor.dev/human-attention-and-human-effort/

Do not submit huge pull requests - we won't review them. Break down large features into chunks, and submit separate changes separately. AI agents will often do this without you realizing (an obvious sign that it was vibe-coded).

Please don't let that put you off submitting changes - but also please understand that in a world where anyone can ask an agent to write thousands of lines of code in minutes, we cannot accept all of that code into our codebase at the same rate while keeping it maintainable and at a high quality.

### Sign off

After you make a PR a comment from @CLAassistant will appear asking you to sign the CLA. This will link a page to allow you to confirm that you have read and agreed to the CLA by signing in with GitHub.

Alternatively, you can sign off before opening a PR by going to https://cla-assistant.io/coredevices/libpebble3.

We accept contributions under a legally identifiable name, such as your name on government documentation or common-law names (names claimed by legitimate usage or repute). Unfortunately, we cannot accept anonymous contributions at this time.
